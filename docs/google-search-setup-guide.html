Search setup

Make Esports Community discoverable on Google

This guide covers the owner-side steps after the site code has been prepared: verify ownership, submit the sitemap, inspect important pages, and keep publishing useful public content.

Code side Metadata, sitemap, robots, icons, and structured data are handled by the app.
Your side Set the public domain, verify Search Console, and submit the sitemap.
Timeline Indexing is not instant. Expect days to weeks, sometimes longer.

1. Confirm the final public URL

Decide the one canonical domain Google should index. Use the real HTTPS public domain, not a temporary CranL URL if you plan to use a custom domain later.

Environment variable Recommended value
EWC_DASHBOARD_PUBLIC_URL https://your-domain.example
BETTER_AUTH_URL Use the same public HTTPS origin as above.
EWC_GOOGLE_SITE_VERIFICATION Set this after Google gives you an HTML tag verification token.

2. Verify ownership in Google Search Console

  1. Open Google Search Console.
  2. Click Add property.
  3. Prefer Domain property if you control DNS. This verifies every subdomain and protocol with one DNS TXT record.
  4. If DNS is annoying, use URL prefix for the exact public URL and choose HTML tag.
  5. Copy only the token inside content="..." into EWC_GOOGLE_SITE_VERIFICATION on CranL, then redeploy.
  6. Click Verify in Search Console.

3. Submit the sitemap

After deployment, open these URLs in your browser first:

  • https://your-domain.example/robots.txt
  • https://your-domain.example/sitemap.xml
  • https://your-domain.example/icon.svg

In Search Console, go to Sitemaps and submit: https://your-domain.example/sitemap.xml

4. Request indexing for the important pages

Use the URL Inspection tool in Search Console for:

  • /
  • /games
  • /news
  • /tournaments
  • /predictions
  • One strong game page, for example /games/valorant.
  • One published article page.

For each page, click Test live URL, then Request indexing.

5. What helps ranking most

  • Publish real public content: news posts, game pages, and tournament pages that users actually read.
  • Use clear titles: include game names, event names, or team names naturally.
  • Link to the site: Discord announcements, X posts, community profiles, GitHub README, and partner pages help Google discover it.
  • Keep pages stable: avoid changing the domain or URL structure after Google starts indexing.
  • Avoid thin pages: empty pages can be crawled, but strong pages are more likely to be indexed and shown.

6. Quick checks after deployment

  1. Search Google for site:your-domain.example.
  2. Check Search Console Pages for indexing issues.
  3. Check Search Console Sitemaps for successful sitemap fetch.
  4. Run a rich result test for an article: Rich Results Test.
  5. Use PageSpeed Insights for performance and Core Web Vitals: PageSpeed Insights.
